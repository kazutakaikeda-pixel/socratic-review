# レビューAI

提案書・定例会資料の品質を高めるためのAIレビューツール。
Claude Codeのスラッシュコマンド `/review` として動作する。

## 使い方

prj_TOYOTAディレクトリでClaude Codeを起動し、以下のコマンドを実行:

```
/review レビューしたいテキストをここに貼り付け
```

ファイルを指定する場合:
```
/review @path/to/file.md
```

## レビュー観点

`ai_review/prompts/` 配下の `.md` ファイルがレビュー観点として読み込まれる。

| ファイル | 観点 |
|---|---|
| `01_target.md` | 的（ターゲット）の評価 |
| `02_content.md` | 球（コンテンツ）の評価 |
| `03_logic_structure.md` | 論理構造（ピラミッド原則） |
| `04_writing_quality.md` | 文書表現 |
| `05_executive_review.md` | 総合評価（外資コンサルパートナー視点） |
| `06_ishikawa_review.md` | 石川さん視点（エンジニア・デリバリー担当）|
| `07_maeda_review.md` | 前田さん視点（パートナー・資料設計者）|

## レビュー観点の追加方法

1. `ai_review/prompts/` に新しい `.md` ファイルを追加する
2. ファイル名は `06_観点名.md` のように番号プレフィックスをつける（実行順序に反映）
3. 以下のフォーマットで記載する:

```markdown
# 観点名

## 概要
何を評価するか

## チェック項目
- チェック項目1
- チェック項目2

## 評価基準
- ○: 基準
- △: 基準
- ×: 基準
```

4. 次回の `/review` 実行時から自動的にレビューに含まれる

## 過去の指摘例

`ai_review/examples/` に過去の指摘例・修正例を `.md` ファイルとして追加可能。
プロンプトに組み込むことで、レビューの精度向上に活用できる（今後対応予定）。
